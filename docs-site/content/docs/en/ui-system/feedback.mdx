Feedback components make background operations legible: provider state, proxy state, OAuth progress, update checks, empty results, loading, and usage refreshes.

## Review states

- Success, warning, error, and neutral variants.
- Alert anatomy: icon, title, description, rich content, action slot, and default glyph behavior.
- Badge anatomy and interaction: variants, status dots, inline icon content, spinner content, link-like badges, command activation, focus-visible, and keyboard activation.
- Avatar anatomy and interaction: image loading status, delayed fallback, error fallback, status badge, AvatarGroup overlap, and AvatarGroupCount overflow members.
- Long text wrapping.
- Loading and empty states.
- Empty-state primary and secondary actions, including loading and disabled trigger suppression.
- Reduced-motion expectations.
